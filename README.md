<h1>Práctica #CARAS_M41 de mi curso de FrontEnd de EBAC</h1>
<h2>📌Descripción del proyecto</h2>
Este proyecto es una aplicación web desarrollada como parte del programa de <b>Front-End de EBAC</b>, enfocada en construir una interfaz moderna, responsiva e interactiva.
La aplicación simula un sitio web funcional orientado a la experiencia de usuario, aplicando buenas prácticas de desarrollo frontend y despliegue continuo.

---

<h2>❗Problema que resuelve</h2>
Muchos proyectos iniciales de desarrollo web fallan en:
<br><br>
<ul>
  <li>Falta de diseño responsivo.</li>
  <li>Experiencia de usuario deficiente.</li>
  <li>Ausencia de interactividad real.</li>
  <li>No contar con un flujo completo de desarrollo (código → deploy).</li>
</ul>  
Este proyecto resuelve esos problemas al:
<br><br>
<ul>
  <li>Implementar una interfaz adaptable a distintos dispositivos.</li>
  <li>Integrar lógica con JavaScript para interacción del usuario.</li>
  <li>Simular un flujo real de producto frontend.</li>
  <li>Desplegar en la nube para acceso público.</li>
</ul>

---

## 🛠️ Stack tecnológico
El desarrollo se realizó utilizando:
<ul>
  <li><b>HTML5</b> → Estructura semántica.</li>
  <li><b>CSS3 / Flexbox / Grid</b> → Diseño responsivo.</li>
  <li><b>JavaScript (ES6+)</b> → Interactividad y manipulación del DOM.</li>
  <li><b>Git & GitHub</b> → Control de versiones.</li>
  <li><b>Netlify</b> → Deploy y hosting.</li>
</ul>

---

## ⚙️ Decisiones técnicas
<ol>
  <li><b>Arquitectura simple (Frontend puro).</b></li>
  <br>
  Se optó por una arquitectura sin backend para:
  <br><br>
  <ul>
    <li>Reducir complejidad.</li>
    <li>Enfocar en fundamentos frontend.</li>
    <li>Mejorar tiempos de carga.</li>
  </ul>
  <br>
  <li><b>Uso de JavaScript vanilla.</b></li>
  <br>
  En lugar de frameworks:
  <br><br>
  <ul>
    <li>Permite entender el DOM a profundidad.</li>
    <li>Facilita debugging.</li>
    <li>Mejora la base técnica.</li>
  </ul>
  <br>
  <li><b>Diseño responsivo desde el inicio.</b></li>
  <br>
  <ul>
    <li>Mobile-first approach.</li>
    <li>Uso de Flexbox/Grid.</li>
    <li>Adaptación a diferentes resoluciones.</li>
  </ul>
  <br>
  <li><b>Deploy en Netlify.</b></li>
  <br>
  <ul>
    <li>CI/CD automático desde repositorio.</li>
    <li>Preview de cambios.</li>
    <li>Hosting optimizado para sitios estáticos.</li>
</ol>
